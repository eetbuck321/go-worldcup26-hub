# go-worldcup26-hub

## 简介

go-worldcup26-hub 是一个用于归档和发布独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，旨在以结构化方式保存一系列自包含的网页资源，便于后续查阅、分发或离线使用。

## 目录说明

```
go-worldcup26-hub/
├── pages/          # 存放所有独立 HTML 页面
├── assets/         # 页面引用的公共资源（如图片、样式、脚本）
├── docs/           # 相关文档或说明文件
└── README.md       # 本文件
```

- **pages/**：每个子目录或文件对应一个独立页面，页面内所有资源尽量自包含。
- **assets/**：若页面需要共享资源，可统一放置于此，并在页面中通过相对路径引用。
- **docs/**：存放与项目相关的额外文档，如归档说明、使用指南等。

## 页面归档说明

- 所有 HTML 页面均为独立文件，不依赖外部服务器或数据库。
- 页面命名遵循“主题-日期”或“主题-序号”格式，便于识别。
- 每个页面在归档时，建议在 `<head>` 中添加元数据注释（如归档日期、原始来源描述），以保持可追溯性。
- 页面中可能包含外部资源（如字体、图标），建议尽量内联或转为本地引用，以增强离线可用性。

## 维护说明

- 欢迎提交新页面归档，请将 HTML 文件放入 `pages/` 目录，并确保文件格式正确、无外部失效链接。
- 若需修改或更新已有页面，请提交 Pull Request，并在描述中说明变更原因。
- 本仓库不主动推广、不承诺更新频率，仅作为静态页面存档使用。
